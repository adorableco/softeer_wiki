## 📒 1. 리뷰

### 오늘 한 작업

1. 글쓰기에 이미지 첨부가 가능하도록 구현하고 있음.
2. char[]로 받던 HTTP Request 데이터를 byte[]로 받는 것으로 변경함.
3. JVM 학습을 진행함. (어렵다...)
4. 기획 요구사항 정의서 피드백을 보냄.

### 작업 과정에서의 배운 것

- byte 배열을 문자열로 변환했다가 다시 byte 배열로 바꾸면 데이터 손실이 발생할 수 있음.
  - 근데 아직 이유는 못찾음!!!
- byte[] 를 처리하면서 **String과 char[] 중에서 String을 사용하는 이유는 제공하는 기능이 많기 때문**이라고 배운 것이 체감됐음.
  - indexOf(), subString() 등의 기능을 수행하는 걸 모두 직접 구현해야하는 번거로움이 있었음.

### 어려웠던 점

- 이미지가 제대로 들어오는지 확인하기 위해 `회원가입 -> 로그인 -> 글쓰기`를 모두 거쳐야해서 테스트 과정에서 엄청난 비효율이 있다고 느낌.
  - mock 사용자를 만들어둔다거나 다른 방식으로 테스트의 효율성을 높여야 함.

## 📒 2. 회고 (KPT)

### 🤸‍♂️ Keep

1. 요구사항 정의서를 꼼꼼히 읽고 나중에 문제가 일어날 수도 있는 애매한 점을 정리할 수 있도록 피드백 드림. 
   - e.g) 사용자가 작물을 아예 등록하지 않는 경우도 있는가?, 도로명 주소가 아닌 **지번 주소**를 사용하는 이유가 따로 있는가?
2. 오랫동안 해결하지 못하면 주변 사람에게 도움을 요청함.
   - 대부분 비슷한 문제를 겪기 때문에 문제를 해결하기 위한 힌트를 얻을 수 있었음.
   - AI에게 답을 요구하는게 아니라 대화하면서 깨닫다보니 기억에도 더 잘 남는 것 같음.


### 🪄 Problem

1. 문제 해결을 하고 나면 그 문제와 해결법에 대해 제대로 짚고 넘어가야하는데 해결에만 급급하다고 생각함.


### 🎯 Try

1. 주변 사람들에게 과제 해결에 도움이 될 수 있는 사람이 되기... 질문에 열려 있는 사람되기
- 오늘 팀원분께 질문을 많이 했는데 내 마음가짐이 바뀐 것도 있지만 팀원이 먼저 질문도 많이 하시고 내가 하는 질문을 잘 들어줬기 때문도 있는 것 같음.
- 나도 주변 사람이 스스럼없이 질문할 수 있도록, 잘 몰라도 최선을 다해 답하려고 함. (잘 모르는 부분이면 잘못된 정보를 전달할까봐 사린 부분이 있는데 그래도 아는 선까진 얘기해봐야겠음.)
