## 7/23 Daily Retrospect

## 📒 1. 리뷰

### 오늘 한 작업

1. 글쓰기에 이미지를 첨부할 수 있도록 기능을 고도화함.
2. 웹페이지에서 사용자가 게시한 이미지가 보이도록 구현함.
3. 게시물 목록 조회에서 게시물 개수가 0개일 경우 발생하는 오류를 해결함.
4. 마이페이지에서 정보를 업데이트하는 기능을 구현함.
5. 405 METHOD_NOT_ALLOWED 오류 발생 시 에러페이지로 리디렉션되도록 구현함. 
6. UserDatabase 중 일부 기능을 **JDBC + h2** 를 사용해 처리하는 것으로 변경함.
7. JDBC 연결을 위한 JDBCUtil을 구현함.

### 작업 과정에서의 배운 것

- `Objects.hash(path, method)` : 여러 값을 기반으로 하나의 해시코드를 계산해주는 유틸리티 메서드
- html form 은 `PUT`, `DELETE`  메서드 사용이 불가능함
    - `<input type="hidden" name="_method" value="PUT"/>` 이런 식으로 따로 명시해주면 된다고는 하는데 타임리프나 다른 템플릿 엔진을 사용하는 경우에 이를 자동으로 인식해주는 것 같음

### 어려웠던 점

- 이미지 바이트 배열이 제대로 들어왔는지 확인하기가 굉장히 어려웠음 (사람이 읽을 수 있는 문자열이 아니라서)
    - 인텔리제이 디버깅에 **‘클립보드와 비교’** 기능이 있으니 잘 활용하자!
